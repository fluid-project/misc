In this example, the two models are rendered separately using autobinding. modelChanged event listeners have been attached to the two changeAppliers. Each time a model changes, a log message displays the new model values. The two models are getting mixed up.

To replicate the problem:

  1. Open this file in Firefox with Firebug installed.
  2. Watch the Firebug console for log messages.
  3. Edit various fields.

You will see that when the second line of fields is edited, the model ends up containing the values in the first line.
